# Ausführungs-Roadmap 2026-06-24 (autonom)

60 offene Tickets (Tag `morgen`). Reihenfolge: **sicher + hoher Wert zuerst**, kalibrierte
Engine-Tickets nur mit Findings-Gleichheitsprüfung, Max-Input/externe ganz am Ende markiert.
Bei „go" arbeite ich Welle für Welle ab. Es ist eine **priorisierte Queue** — wir kommen so weit
wie wir kommen; das Wichtigste zuerst.

## Legende (Autonomie)
- 🟢 **autonom** — bauen, verifizieren, deployen ohne Rückfrage.
- 🟡 **autonom + Verifikation** — ändert Engine/kalibriertes Verhalten → vorher/nachher Findings-
  Signatur über alle Projekte diffen (wie beim Gitter-Index); bei Abweichung NICHT silently ändern,
  sondern Diff surfacen. Max-Regel: Engine-Kalibrierung nicht ohne Beleg ändern.
- 🔵 **braucht Max-Input/Entscheidung** — ich bereite vor + lege Entscheidungsvorlage hin.
- ⚫ **extern-blockiert** — Dritte nötig (Behörde/Lizenz/Reputation-Recategorization); ich bereite den
  Antrag/Text vor, abschließen kann nur Max.

## Arbeitsprotokoll je Welle
1. Dach-Ticket je Welle (`kanban add`), Sub-Tickets `in_progress`.
2. Pro Ticket: bauen → Tests/tsc/Build grün → committen → deployen (api/worker direkt, FE via Hub-
   Vendoring) → live verifizieren → `kanban log`/`done`.
3. Engine-Tickets (🟡): Baseline-Findings-Signatur ziehen → ändern → diffen → nur bei Gleichheit live.
4. Je Welle eine kurze Zusammenfassung an Max. Brain-Notizen bei nicht-trivialen Fixes.
